## 1. 产品概述

**音频编辑器**是 Original Character Maker v1.5.0 推出的内置音频 waveform（波形）编辑工具。它基于浏览器原生 **Web Audio API** 构建，无需安装任何插件或外部音频软件，即可在浏览器中完成音频的导入、剪辑、效果处理和导出全流程。

### 核心能力

| 类别 | 功能 |
|------|------|
| **导入** | MP3、WAV、OGG、FLAC、M4A、AAC、WebM 等主流格式 |
| **波形显示** | 可交互的 Canvas 波形图，支持缩放、平移、框选 |
| **剪辑操作** | 修剪、分割、删除、复制、反转、淡入淡出、标准化、单声道混合 |
| **实时效果** | 音量、声像、速度、音调、循环、EQ、压缩器、混响 |
| **渲染效果** | 降噪、标准化（在导出阶段离线渲染） |
| **导出** | WAV（8/16/32-bit）、WebM/Opus、OGG/Opus、MP3、MP4/AAC |
| **辅助功能** | 撤销/重做、操作日志、导出历史、响应式布局 |

> ⚠️ **重要前提**：音频编辑器完全在浏览器端运行，所有音频数据都保存在本地内存中，**不会上传到服务器**。这意味着你的隐私得到完全保护，但也意味着页面刷新后所有未导出的工作都会丢失。

## 2. 快速开始（3 步上手）

### 第 1 步：打开工具

在 Original Character Maker 首页点击 **「音频编辑器」** 按钮，或在工作流启动弹窗中选择音频编辑器图标进入。

### 第 2 步：导入音频

进入编辑器后，你会看到一个大的导入区域，显示 🎵 图标和文字 **「Import Audio」**。点击该区域，在文件选择器中选择你的音频文件（支持拖放）。导入成功后，波形会立即显示在画布上。

### 第 3 步：编辑并导出

1. 在波形上 **按住鼠标左键拖动** 框选一个区域；
2. 点击工具栏上的 **「✂ Trim」** 按钮，音频会被裁剪为选中部分；
3. 在下方 **「💾 Export」** 区域选择格式（默认 WAV 16-bit），点击 **「Export」**；
4. 文件会自动下载到你的电脑。

## 4. 导入音频

### 4.1 支持的格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| MP3 | `.mp3` | 最常用，兼容性最好 |
| WAV | `.wav` | 无损 PCM，推荐用于后期处理 |
| OGG | `.ogg` | 开源格式，压缩率高 |
| FLAC | `.flac` | 无损压缩，保留完整音质 |
| M4A | `.m4a` | Apple 常用格式（AAC 编码） |
| AAC | `.aac` | 高级音频编码 |
| WebM | `.webm` | Google 容器格式 |

> 💡 **提示**：浏览器通过 `AudioContext.decodeAudioData()` 解码音频。只要浏览器支持解码的格式，都可以导入。绝大多数现代浏览器都支持 MP3 和 WAV。

### 4.2 导入方式

**方式 A：首次导入**

进入音频编辑器页面后，点击中央的大导入区域（显示 🎵 图标），在弹出的文件选择器中选取音频文件。

**方式 B：更换文件**

如果已经加载了一个音频，点击工具栏最右侧的 **「📁 New File」** 按钮，可以重新导入另一个文件。这会**丢弃当前所有编辑内容**，请确保已导出需要保存的文件。

**方式 C：拖放（如应用支持）**

部分浏览器和系统支持将文件直接拖放到导入区域。松开鼠标后文件即开始加载。

### 4.3 导入后的信息

导入成功后，工具栏会显示以下信息：

```
文件名.mp3 · 2:35.400
```

同时日志面板会记录：

```
[SUCCESS]: Import complete: 2ch, 44100Hz, 2:35.400
```

- `2ch` = 双声道（立体声）
- `44100Hz` = 采样率
- `2:35.400` = 音频总时长

### 4.4 导入失败的可能原因

- 文件不是有效的音频文件（比如改扩展名的图片/视频）
- 文件使用了浏览器不支持的编码格式
- 文件过大导致浏览器内存不足
- 音频文件已损坏

详见 [错误代码表](#13-错误代码表) 中的 `IMPORT_FAILED` 和 `DECODE_ERROR`。

## 5. 波形查看器详解

波形查看器是整个音频编辑器的核心交互区域。它基于 HTML5 Canvas 渲染，显示音频的振幅随时间变化的图形。

### 5.1 波形元素说明

| 视觉元素 | 颜色/样式 | 含义 |
|----------|-----------|------|
| 波形线条 | 主色调（默认紫色/青色） | 音频振幅，越高表示音量越大 |
| 中心线 | 浅灰色虚线 | 0 振幅基准线 |
| 背景网格 | 极淡的横线 | 辅助对齐参考 |
| 紫色半透明遮罩 | `rgba(124,92,255,0.25)` | **已选中的时间区域** |
| 红色竖线 | `#f45a5a` | **播放头**，指示当前播放位置 |

### 5.2 鼠标操作

| 操作 | 效果 |
|------|------|
| **左键拖动** | 框选一个时间区域（显示紫色遮罩） |
| **Shift + 左键拖动** | 平移波形（左右滚动查看不同部分） |
| **单击** | 将播放头移动到点击位置 |
| **Ctrl/⌘ + 滚轮** | 缩放波形（放大看细节，缩小看全局） |
| **普通滚轮** | 左右平移波形 |

### 5.3 缩放（Zoom）

- **缩放范围**：`1x` 到 `50x`
- `1x`：显示整个音频的全貌
- `50x`：最大放大，适合精确到毫秒级别的编辑
- 当前缩放倍率显示在时间轴信息栏右侧

### 5.4 选择（Selection）

框选后，选择区域下方会显示：

```
Selection: 0:05.200 – 0:12.800 (0:07.600)
```

格式为：`开始时间 – 结束时间 (选中时长)`

很多编辑操作（Trim、Delete、Duplicate）都**依赖于选中区域**。如果没有选中任何区域，这些按钮会处于禁用状态（灰色）。

> 💡 **技巧**：如果选错了区域，只需在波形上重新框选即可覆盖之前的选择。点击「Stop」按钮不会取消选择，但点击波形空白处（不拖动）会将播放头移到该位置。

### 5.5 播放头

红色竖线指示当前播放位置。播放时它会实时移动。播放结束后，播放头停在音频末尾。

## 6. 播放控制

播放控制按钮位于工具栏第一行，从左到右依次是：

### 6.1 按钮说明

| 按钮 | 功能 | 快捷键 |
|------|------|--------|
| **▶ Play / ⏸ Pause** | 开始播放或暂停 | `Space` |
| **⏹ Stop** | 停止播放并将播放头归零 | — |
| **↩ Undo** | 撤销上一步编辑操作 | `Ctrl+Z` |
| **↪ Redo** | 重做被撤销的操作 | `Ctrl+Shift+Z` |

### 6.2 实时效果监听

音频编辑器支持**实时效果预览**：你在效果面板中调整的任何参数（音量、EQ、混响等）都会在播放时**即时生效**，无需等待渲染。

实时效果的信号链如下：

```
音频源 → 3段EQ → 压缩器 → [干声增益 + 混响湿声] → 主增益 → 声像 → 扬声器
         ↓          ↓              ↓
       低频增益   阈值/比率      房间大小/衰减
       中频增益   攻击/释放      干湿比
       高频增益
```

> ⚠️ **注意**：降噪（Noise Reduction）和导出时的标准化是**渲染效果**，不会在实时播放中体现。它们只在导出阶段离线处理。

### 6.3 循环播放（Loop）

在「⚡ Speed & Pitch」卡片中开启 **Loop** 开关后：

- 如果**没有选择区域**：整个音频会循环播放
- 如果**有选择区域**：只循环播放选中的部分

### 6.4 反向播放（Reverse）

开启 **Reverse** 开关后，播放时会倒着播放音频。这在实时预览和导出时都会生效。

## 7. 编辑操作详解

所有编辑操作都会**永久修改**当前音频数据（可通过 Undo 撤销）。操作成功后，波形会立即重绘。

### 7.1 修剪（Trim）

**作用**：删除选中区域之外的所有音频，只保留选中部分。

**使用步骤**：
1. 在波形上框选要保留的区域
2. 点击工具栏 **「✂ Trim」**
3. 音频被裁剪，播放头归零

**边界检查**：
- 如果没有选择区域，按钮禁用
- 如果选择的开始时间 ≥ 结束时间，或开始时间 ≥ 音频总时长，操作会被拒绝并提示 "Invalid selection for trim"

### 7.2 分割（Split）

**作用**：在选择区域的**起始位置**将音频切开，但**保留两部分**，只是在此处做一个标记。

> 💡 **实际行为**：Split 操作在内部将音频切成两段后立即重新拼接，因此从听觉和视觉上看音频没有变化。它的主要用途是在特定时间点做一个可撤销的「切割点」，为后续的 Delete 或更复杂的手动编辑做准备。

**使用步骤**：
1. 框选一个区域（分割点取该区域的起始时间）
2. 点击工具栏 **「✂ Split」**
3. 日志会记录：`Split at 0:xx.xxx`

### 7.3 删除（Delete）

**作用**：删除选中的时间区域，将前后两段音频拼接起来。

**使用步骤**：
1. 框选要删除的区域
2. 点击工具栏 **「🗑 Delete」**
3. 选中部分被移除，播放头移到删除位置

**边界检查**：
- 如果选择的开始时间 ≤ 0 且结束时间 ≥ 音频总时长（即全选），操作会被拒绝，提示 "Cannot delete entire audio"
- 这是为了防止误操作导致音频被完全清空

### 7.4 复制（Duplicate）

**作用**：将选中的区域复制一份，并**追加到音频末尾**。

**使用步骤**：
1. 框选要复制的区域
2. 点击工具栏 **「📋 Duplicate」**
3. 选中部分被复制并追加到原音频尾部

**示例**：原音频 10 秒，复制 2-5 秒区域，结果音频变为 13 秒，其中 10-13 秒是 2-5 秒的内容。

### 7.5 反转（Reverse）

**作用**：将整个音频倒放。

**使用步骤**：
1. 点击工具栏 **「🔀 Reverse」**
2. 音频被完全反转，波形会倒过来显示

> 💡 **与 Reverse 开关的区别**：工具栏的 Reverse 是**编辑操作**（永久修改音频数据），而效果面板中的 Reverse 开关是**实时效果**（只在播放时倒放，不修改底层数据）。如果你需要导出倒放的音频，可以使用编辑操作，或者开启 Reverse 开关后导出。

### 7.6 淡入淡出（Fade）

**作用**：在音频的开头和/或结尾应用音量渐变。

**使用步骤**：
1. 在「📉 Fade」效果卡片中设置 **Fade In** 和/或 **Fade Out** 时间（0-10 秒）
2. 点击工具栏 **「📉 Fade」**
3. 音频被永久修改，开头和/或结尾出现音量渐变

> 💡 **技巧**：Fade In/Out 参数同时影响实时播放和编辑操作。也就是说，你可以在效果面板中调好参数，先实时试听，满意后再点击 Fade 按钮永久应用。

### 7.7 标准化（Normalize）

**作用**：将音频的峰值提升到最大可能幅度（0 dBFS），使音量最大化，同时不削波。

**使用步骤**：
1. 点击工具栏 **「📈 Normalize」**
2. 音频峰值被自动调整到 99.9%（留 0.1% 余量防止数字削波）

**边界检查**：
- 如果音频已经是静音（所有采样值为 0），标准化不会做任何事
- 如果音频峰值已经 ≥ 99.9%，也不会处理

### 7.8 单声道混合（Mono）

**作用**：将立体声（多声道）音频混合为单声道。

**使用步骤**：
1. 点击工具栏 **「🔊 Mono」**
2. 如果音频是多声道的，会被混合为 1 个声道
3. 如果音频本来就是单声道，不会有变化

**计算公式**：每个采样点的值 = 所有声道采样值的平均值

## 8. 效果器系统

音频编辑器提供 **7 组效果参数**，分为两类：

| 类型 | 效果 | 应用时机 |
|------|------|----------|
| **实时效果** | 音量、声像、速度、音调、循环、反向、EQ、压缩器、混响 | 播放时即时生效 |
| **渲染效果** | 降噪、标准化 | 仅在导出时离线处理 |

### 8.1 音量与声像（Volume & Pan）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Volume** | 0% ~ 200% | 100% | 主音量增益，200% = 两倍音量 |
| **Mute** | On / Off | Off | 静音开关，优先级高于 Volume |
| **Pan (L/R)** | -100 ~ +100 | 0 | 声像定位，-100=极左，+100=极右，0=居中 |

### 8.2 速度与音调（Speed & Pitch）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Speed** | 25% ~ 400% | 100% | 播放速度，影响时长和音调（非独立） |
| **Pitch** | -1200 ~ +1200 cents | 0 | 音高偏移，±100 cents = 半音，±1200 = 八度 |
| **Reverse** | On / Off | Off | 倒放开关 |
| **Loop** | On / Off | Off | 循环播放开关 |

> ⚠️ **Speed 与 Pitch 的关系**：Speed 改变的是 `playbackRate`，这会同时改变播放速度和音调（就像磁带快放/慢放）。Pitch 参数通过 `detune` 独立调整音高。两者可以组合使用，实现变速不变调或变调不变速的效果。

### 8.3 淡入淡出（Fade）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Fade In** | 0 ~ 10.0 秒 | 0 | 开头音量从 0 渐变到 100% 的时间 |
| **Fade Out** | 0 ~ 10.0 秒 | 0 | 结尾音量从 100% 渐变到 0 的时间 |

渐变曲线为**线性渐变**（Linear Fade）。

### 8.4 3 段均衡器（EQ）

基于 Web Audio API 的 `BiquadFilterNode`：

| 参数 | 范围 | 默认值 | 频段 | 滤波器类型 |
|------|------|--------|------|------------|
| **Low Gain** | -12 dB ~ +12 dB | 0 dB | 200 Hz | Low-shelf |
| **Mid Gain** | -12 dB ~ +12 dB | 0 dB | 1000 Hz | Peaking |
| **High Gain** | -12 dB ~ +12 dB | 0 dB | 5000 Hz | High-shelf |

- **Low-shelf**：提升或衰减 200 Hz 以下的所有频率
- **Peaking**：以 1000 Hz 为中心频率做钟形提升/衰减
- **High-shelf**：提升或衰减 5000 Hz 以上的所有频率

### 8.5 动态压缩器（Compressor）

基于 Web Audio API 的 `DynamicsCompressorNode`：

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Threshold** | -60 dB ~ 0 dB | -24 dB | 压缩开始触发的阈值 |
| **Ratio** | 1:1 ~ 20:1 | 4:1 | 压缩比，超过阈值的信号按此比例压缩 |
| **Attack** | 0 ~ 100 ms | 3 ms | 压缩器开始响应的时间 |
| **Release** | 0 ~ 500 ms | 50 ms | 压缩器停止压缩后的恢复时间 |

**用途**：控制动态范围，让安静的段落更清楚，同时防止大声段落削波。

### 8.6 混响（Reverb）

使用算法生成的脉冲响应（Impulse Response）配合 `ConvolverNode`：

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Room Size** | 0% ~ 100% | 30% | 虚拟房间大小，决定混响时长（0.5s ~ 4s） |
| **Damping** | 0% ~ 100% | 50% | 高频衰减程度，越高高频消失越快 |
| **Wet/Dry** | 0% ~ 100% | 20% | 混响信号与干声的比例，0%=只有原声，100%=只有混响 |

> 💡 **技术细节**：混响脉冲响应是实时生成的随机衰减白噪声，长度 = `(0.5 + roomSize/100 * 3.5) * sampleRate` 个采样点。因此 Room Size 为 0% 时，混响时长约 0.5 秒；100% 时约 4 秒。

### 8.7 降噪（Noise Reduction）

**渲染效果**——只在导出时应用，实时播放中不可监听。

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Noise Reduction** | 0% ~ 100% | 0% | 降噪强度，0%=不降噪 |

**算法原理**：软阈值门限（Soft Noise Gate）

1. 计算阈值：`threshold = (intensity / 100) * 0.02`
2. 对于每个采样值 `v`：
   - 如果 `|v| < threshold`：将该采样乘以 `|v| / threshold`（ softer attenuation）
   - 如果 `|v| ≥ threshold`：保持不变

> ⚠️ **副作用**：降噪强度过高会削减微弱的音频细节（如呼吸声、轻柔的乐器泛音），建议适度使用。

### 8.8 标准化（Normalize）——渲染开关

效果面板中的 **Normalize** 开关与工具栏的 Normalize 按钮不同：

- **工具栏按钮**：立即对当前音频做标准化（编辑操作）
- **效果面板开关**：在**导出时**自动做标准化（渲染效果）

两者可以同时使用，但通常没必要。

### 8.9 重置所有效果

点击工具栏上的 **「🔄 Reset FX」** 按钮，所有 21 个效果参数会瞬间恢复为默认值。

## 11. 历史与撤销系统

### 11.1 工作机制

每次执行编辑操作（Trim、Delete、Duplicate、Reverse、Fade、Normalize、Mono）时，系统会将当前音频的完整副本推入历史栈。

- **Undo**：回退到上一个历史状态
- **Redo**：前进到下一个历史状态
- 执行新的编辑操作后，Redo 栈会被清空

### 11.2 限制

- 历史栈**只保存音频数据快照**，不保存效果参数（音量、EQ 等实时效果的状态不会被记录在历史中）
- 历史数据保存在内存中，页面刷新后全部丢失
- 导入新文件会**重置整个历史栈**

### 11.3 日志提示

执行 Undo/Redo 时，日志会显示：

```
[INFO]: Undo: restored step 3
[INFO]: Redo: restored step 4
```

## 14. 故障排除指南

### 问题：导入后波形不显示

**可能原因**：
- 音频文件是静音或音量极低
- Canvas 渲染异常

**解决方法**：
1. 确认音频在其他播放器中有声音
2. 尝试缩放波形（Ctrl + 滚轮）看是否能看到线条
3. 调整窗口大小触发重绘
4. 刷新页面重新导入

### 问题：播放没有声音

**可能原因**：
- 浏览器自动播放策略阻止了 AudioContext
- Mute 开关被开启
- Volume 被设为 0%
- 系统音量静音

**解决方法**：
1. 点击 Play 按钮时，应用会自动尝试恢复被挂起的 AudioContext
2. 检查效果面板中的 Mute 开关是否关闭
3. 检查 Volume 是否大于 0%
4. 确认系统音量和浏览器标签页未被静音

### 问题：导出后的文件音质变差

**可能原因**：
- 选择了 8-bit WAV（音质明显较差）
- 使用了降噪（会削减细节）
- MediaRecorder 压缩格式的编码质量由浏览器决定，无法调整

**解决方法**：
1. 选择 WAV 16-bit PCM 或 32-bit Float 以获得最佳音质
2. 降低降噪强度或关闭降噪
3. 如果必须使用压缩格式，建议导出 WAV 后用专业工具压缩

### 问题：导出格式下拉菜单选项很少

**可能原因**：
- 当前浏览器不支持 MediaRecorder，或只支持少数 MIME 类型

**解决方法**：
- WAV 格式（8/16/32-bit）是原生编码，**所有浏览器**都支持
- 如需 MP3/OGG/WebM，请使用 Chrome、Edge 或 Firefox 的最新版本
- Safari 对 MediaRecorder 的支持较有限

### 问题：操作后内存占用很高

**可能原因**：
- 长音频文件（> 30 分钟）会产生大量 AudioBuffer 数据
- 历史栈中保存了多个音频快照
- 导出记录太多

**解决方法**：
1. 对于极长音频，建议先用外部工具剪辑到需要的长度再导入
2. 定期清理导出记录（Results 面板中删除不需要的文件）
3. 如果浏览器提示内存不足，刷新页面重新开始

### 问题：Undo/Redo 行为异常

**可能原因**：
- 效果参数（如 Volume、EQ）不是历史的一部分，撤销不会改变它们
- 导入新文件会清空历史

**说明**：
- 这是设计上的行为。历史栈只记录音频数据的变更，不记录实时效果参数。撤销后如果听到不同的声音，可能是因为效果参数当前值与执行该操作时不同。

## 15. 常见问题 FAQ

### Q1：音频编辑器需要联网吗？

**不需要**。所有处理都在浏览器本地完成，导入的音频不会上传到服务器，导出也在本地生成。但应用本身是一个网页，首次加载需要联网。

### Q2：支持多轨道编辑吗？

**不支持**。音频编辑器是单轨道编辑器，一次只能处理一个音频文件。如果需要多轨道混音，请使用专业 DAW（如 Audacity、Reaper、FL Studio）。

### Q3：最大支持多大的文件？

没有硬编码限制，但受限于浏览器的可用内存。实际测试表明：

| 浏览器 | 大致安全上限 |
|--------|-------------|
| Chrome | ~ 500 MB 原始音频数据 |
| Firefox | ~ 300 MB |
| Safari | ~ 200 MB |

> 原始音频数据大小 ≈ 时长(秒) × 采样率(Hz) × 声道数 × 4 字节。例如：5 分钟的 44100Hz 立体声 ≈ 101 MB。

### Q4：可以保存项目文件下次继续编辑吗？

**暂时不能**。当前版本没有项目文件格式。建议：
1. 导出为 WAV 保留最高质量
2. 记录你使用的效果参数，下次重新导入后手动设置

### Q5：实时效果和渲染效果有什么区别？

| 对比项 | 实时效果 | 渲染效果 |
|--------|----------|----------|
| 听到效果 | 播放时立即听到 | 导出后才能听到 |
| 修改音频数据 | 否 | 是 |
| 可撤销 | 无需撤销（不修改数据） | 不可逆（已写入导出文件） |
| 例子 | 音量、EQ、混响 | 降噪、导出标准化 |

### Q6：为什么 Speed 和 Pitch 会互相影响？

Web Audio API 的 `playbackRate` 参数本质上是重采样——播放速度变快，频率（音调）也会变高，就像磁带快放一样。Pitch 参数（`detune`）可以独立偏移音高，用于补偿 Speed 带来的音调变化。

**实用公式**：
- 想要 **变速不变调**：调整 Speed 后，将 Pitch 设为反向补偿值（Speed 200% → Pitch -1200 cents）
- 想要 **变调不变速**：保持 Speed = 100%，只调整 Pitch

### Q7：Split 操作后音频听起来没有变化？

这是正常的。Split 在内部将音频切成两段后立即重新拼接，因此听觉上没有变化。它的作用是在时间轴上创建一个「切割点」，方便后续配合其他操作使用（比如先 Split 再 Delete 其中一段）。

### Q8：为什么 EQ 只有 3 段？

当前版本使用 3 个 `BiquadFilterNode` 实现低架、峰值、高架滤波，覆盖低频（200Hz）、中频（1000Hz）、高频（5000Hz）三个主要频段。这已经能够满足大多数基础调整需求。如需更精细的参量 EQ，请使用专业音频软件。

### Q9：混响听起来不够真实？

当前混响使用算法生成的随机脉冲响应，属于简化的数字混响。它适用于基础的空间感增强，但无法与专业卷积混响（使用真实房间录音的 IR）相比。如需出版级混响，建议在导出后用专业插件处理。

### Q10：降噪会损伤音质吗？

会。降噪算法是软门限，它会衰减低于阈值的信号。当降噪强度过高时：
- 会削减微弱的细节（如混响尾音、轻柔的呼吸声）
- 可能产生轻微的「水声」伪影

**建议**：从 10-20% 开始，逐步增加直到噪音被抑制但主要音频内容保持清晰。

## 17. 高级技巧

### 技巧 1：精确选择到毫秒

2. 缓慢拖动选择区域

### 技巧 2：快速试听效果变化

1. 开始播放
2. 边听边调整 EQ、混响等参数
3. 找到满意的效果组合后，再执行导出

### 技巧 3：批量降噪 + 标准化工作流

对于录制的语音素材：
1. 导入音频
2. 设置 Noise Reduction 为 15-25%
3. 开启 Normalize 开关
4. 导出为 WAV 16-bit
5. 获得干净、音量适中的语音文件

### 技巧 4：制作无缝循环

1. 找到音频中适合循环的段落
2. 框选该段落，执行 Trim
3. 开启 Loop 开关试听
4. 微调 Fade In/Out（0.01-0.05 秒）消除接缝爆音
5. 导出

### 技巧 5：利用 Split + Delete 精确删除

如果只想删除一个很短的段落（如咳嗽声、口误）：
1. 放大波形到能看清该段落的程度
2. 框选包含噪音的区域
3. 点击 Delete
4. 前后音频会自动拼接

### 技巧 6：Duplicate 制作回声/重复效果

1. 框选一个短片段（如一个鼓点、一个音符）
2. 点击 Duplicate
3. 该片段会被追加到音频末尾
4. 可以重复多次 Duplicate 制作节奏型

### 技巧 7：结合外部工具

音频编辑器适合快速剪辑和基础处理。对于更复杂的需求：
- **高级混音** → Audacity（免费）、Reaper
- **母带处理** → iZotope Ozone、FabFilter
- **格式转换/批量处理** → FFmpeg
- **从本工具导出 WAV** → 导入专业软件继续处理

## 18. 已知限制与注意事项

### 18.1 当前版本的已知限制

| # | 限制 | 说明 |
|---|------|------|
| 1 | 单轨道 | 不支持多轨道叠加或混音 |
| 2 | 无项目文件 | 无法保存工作进度供下次继续 |
| 3 | 无精确时间输入 | 只能通过鼠标框选，无法直接输入开始/结束时间数值 |
| 4 | 简化的降噪 | 使用门限算法，非谱减法或 AI 降噪 |
| 5 | 简化的混响 | 使用随机 IR，非真实房间采样 |
| 6 | 无可视化频谱 | 只有波形图，没有频谱分析仪或示波器 |
| 7 | 无节拍检测 | 不显示 BPM 或节拍网格 |
| 8 | 内存限制 | 长音频会消耗大量 RAM |
| 9 | 无自动保存 | 页面刷新后所有工作丢失 |
| 10 | 压缩格式质量不可调 | MediaRecorder 的比特率由浏览器控制 |

### 18.2 使用建议

1. **经常导出中间版本**：由于无自动保存，建议每完成一个阶段性操作就导出一次，避免意外丢失。
2. **优先使用 WAV 作为中间格式**：WAV 无损，适合多次编辑后仍保持质量。
3. **降噪适度**：过高的降噪强度会损害音质。
4. **注意浏览器内存**：处理长音频时，如果浏览器变卡，考虑分段处理。
5. **横屏使用**：在平板或手机上使用时，横屏模式能提供更好的波形查看体验。

### 18.3 隐私声明

音频编辑器**不会**将任何音频数据上传到服务器。所有解码、处理、导出操作都在你的设备本地完成。导入的音频文件通过浏览器 File API 读取后直接送入 Web Audio API，不经过网络传输。
