## 1. 产品概述

**音频格式转换器**是 Original Character Maker v1.7.1 内置的专用音频格式转换与处理工具。它基于浏览器原生 **Web Audio API** 构建，无需安装任何插件或外部软件，即可在浏览器中完成音频的格式转换、采样率调整、声道转换、音量调节、速度/音高变换、淡入淡出、降噪和峰值归一化等操作。

### 核心能力

| 类别 | 功能 |
|------|------|
| **导入** | MP3、WAV、OGG、FLAC、M4A、AAC、WebM 等主流格式 |
| **输出格式** | WAV（8/16/24/32-bit PCM）、FLAC（WAV 容器回退）、WebM/Opus、OGG/Opus、MP3、MP4/AAC |
| **采样率** | 保持原始，或转换为 22050 / 44100 / 48000 / 96000 / 192000 Hz |
| **声道** | 保持原始、强制单声道、强制立体声 |
| **音量** | 0% – 200% 幅度调节 |
| **速度** | 25% – 400% 时间伸缩 |
| **音高** | ±1200 cents（±1 个八度） |
| **淡入淡出** | 0 – 10 秒线性渐变 |
| **降噪** | 0 – 100% 强度可调 |
| **峰值归一化** | 自动将最大峰值调整至 99.9% |
| **实时预览** | 转换前后均可通过浏览器原生音频播放器预览 |
| **进度追踪** | 可视化进度条 + 分步骤日志 |
| **工作流日志** | 时间戳记录每一步操作 |

> ⚠️ **重要前提**：音频格式转换器完全在浏览器端运行，所有音频数据都保存在本地内存中，**不会上传到服务器**。这意味着你的隐私得到完全保护，但也意味着页面刷新后所有未下载的结果都会丢失。

### 与音频编辑器的区别

| 特性 | 音频格式转换器 | 音频编辑器 |
|------|--------------|-----------|
| **核心用途** | 批量格式转换、参数调整 | 波形级精确剪辑、效果处理 |
| **波形显示** | ❌ 无 | ✅ 完整 Canvas 波形 |
| **剪辑操作** | ❌ 无 | ✅ 修剪、分割、删除等 |
| **实时效果** | ❌ 无（离线处理） | ✅ 播放时实时生效 |
| **批量处理** | ✅ 单文件处理（未来扩展） | ❌ 单文件 |
| **处理链** | 固定 7 步顺序处理 | 可任意组合、撤销/重做 |
| **互操作** | 可一键跳转到编辑器 | 可独立使用 |

如果你需要对音频进行波形级别的精确剪辑或多效果实时预览，请使用 **音频编辑器**。如果你只需要转换格式、调整采样率或批量应用标准化/降噪等处理，**音频格式转换器**是更快捷的选择。

## 2. 快速开始（3 步上手）

### 第 1 步：打开工具

在 Original Character Maker 首页点击 **「音频格式转换」** 按钮，或在工作流启动弹窗中选择音频转换器图标进入。

### 第 2 步：导入音频

进入转换器后，你会看到 **Source Audio** 区域。点击右上角的 **「Choose File」** 按钮，在文件选择器中选择你的音频文件（也支持拖放到区域内）。导入成功后，区域中会显示原生音频播放器，文件名、大小、声道数、采样率和时长会显示在播放器下方。

### 第 3 步：设置参数并转换

在左侧的 **Conversion Settings** 区域：

1. 选择 **Output Format**（如 WAV 16-bit PCM）
2. 调整需要的参数（采样率、声道、音量等）
3. 点击 **「Convert」** 按钮
4. 等待进度条到达 100%
5. 在右侧 **Result** 区域预览转换结果
6. 点击 **「Download」** 保存到本地

> 💡 全程无需任何服务器交互，所有处理在浏览器内完成。

## 4. 导入音频

### 4.1 支持的格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| MP3 | `.mp3` | 最常用，兼容性最好 |
| WAV | `.wav` | 无损 PCM，推荐用于后期处理 |
| OGG | `.ogg` | 开源格式，压缩率高 |
| FLAC | `.flac` | 无损压缩，保留完整音质 |
| M4A | `.m4a` | Apple 常用格式（AAC 编码） |
| AAC | `.aac` | 高级音频编码 |
| WebM | `.webm` | Google 容器格式 |

> 💡 **提示**：浏览器通过 `AudioContext.decodeAudioData()` 解码音频。只要浏览器支持解码的格式，都可以导入。绝大多数现代浏览器都支持 MP3 和 WAV。

### 4.2 导入方式

**方式 A：点击按钮**

进入转换器页面后，点击 Source Audio 区域右上角的 **「Choose File」** 按钮，在弹出的文件选择器中选取音频文件。如果已经导入了一个文件，按钮文字会变为 **「Replace」**，点击后可更换为其他文件。

**方式 B：拖放（Drag & Drop）**

将音频文件从文件管理器直接拖放到 Source Audio 区域，区域会高亮显示（边框变为强调色实线、背景变为淡色填充），松开鼠标后自动开始加载。如果拖放的文件不是音频格式（MIME 类型不以 `audio/` 开头），会记录日志提示并播放错误音效。

> ⚠️ **注意**：拖放时的 MIME 类型检测依赖操作系统和浏览器。某些文件（如部分 `.wav` 文件）可能被识别为 `application/octet-stream`。如果拖放失败，请使用「Choose File」按钮导入。

### 4.3 导入后的信息

导入成功后，Source Audio 区域下方会显示以下信息：

```
filename.mp3 (2.34 MB) · 2ch, 44100Hz, 3:24.567
```

| 字段 | 说明 |
|------|------|
| `filename.mp3` | 原始文件名 |
| `2.34 MB` | 文件大小 |
| `2ch` | 声道数（1ch = 单声道，2ch = 立体声） |
| `44100Hz` | 采样率 |
| `3:24.567` | 时长（分:秒.毫秒） |

同时，原生 HTML5 `<audio>` 播放器会显示在区域内，你可以点击播放按钮预览原始音频。

### 4.4 导入限制

| 限制项 | 说明 |
|--------|------|
| 单文件大小 | 受浏览器内存限制，建议不超过 500MB |
| 时长 | 理论上无限制，但超过 30 分钟的文件处理时间较长 |
| 格式 | 依赖浏览器解码能力，不支持的格式会报 IMPORT_FAILED 错误 |

## 5. 转换设置详解

### 5.1 Output Format（输出格式）

输出格式下拉框列出所有当前浏览器支持的格式。格式分为两类：

**原生 WAV 格式（无需编码器，所有浏览器都支持）：**

| 选项 | 位深 | 编码 | 说明 |
|------|------|------|------|
| WAV 8-bit PCM | 8-bit | 无符号 PCM | 最低质量，文件最小，复古/8-bit 风格 |
| WAV 16-bit PCM | 16-bit | 有符号 PCM | **推荐**。CD 音质标准，兼容性最佳 |
| WAV 24-bit PCM | 24-bit | 有符号 PCM | 高动态范围，适合母带处理 |
| WAV 32-bit Float | 32-bit | IEEE 浮点 | 最高精度，适合专业音频工作流 |
| FLAC (WAV container) | 16-bit | PCM（WAV 回退） | 由于浏览器无原生 FLAC 编码器，实际输出 WAV 16-bit |

**压缩格式（依赖浏览器 MediaRecorder 支持）：**

| 选项 | 容器 | 编码器 | 兼容性 |
|------|------|--------|--------|
| WebM / Opus | WebM | Opus | Chrome、Firefox、Edge 支持良好 |
| OGG / Opus | OGG | Opus | Firefox 原生支持，Chrome 支持 |
| MP3 | MP3 | MPEG-1 Layer III | Safari 支持，其他浏览器可能不支持 |
| MP4 / AAC | MP4 | AAC | Safari 支持，Chrome 可能不支持 |

> ⚠️ **兼容性提示**：如果你发现某个压缩格式无法转换（进度条卡住或报错），说明当前浏览器不支持该格式的 MediaRecorder。请切换到 WAV 16-bit PCM，这是**所有浏览器都保证支持**的格式。

### 5.2 Sample Rate（采样率）

| 选项 | 采样率 | 适用场景 |
|------|--------|----------|
| Original | 保持源文件采样率 | 默认推荐，不改变时域特性 |
| 22050 Hz | 22050 | 语音、低带宽场景 |
| 44100 Hz | 44100 | CD 音质标准，音乐制作 |
| 48000 Hz | 48000 | 视频制作、专业音频 |
| 96000 Hz | 96000 | 高分辨率音频、母带存档 |
| 192000 Hz | 192000 | 超高分辨率，文件极大 |

> 💡 **关于重采样**：选择非原始采样率时，转换器会使用 `OfflineAudioContext` 进行高质量重采样。重采样过程会同时处理速度变化和音高偏移（如果设置了的话）。

> ⚠️ **注意**：升采样（如从 22050Hz 到 192000Hz）不会增加原始音频中不存在的信息，只会增加文件大小。

### 5.3 Channels（声道）

| 选项 | 说明 |
|------|------|
| Original | 保持源文件的声道数（单声道保持单声道，立体声保持立体声） |
| Mono | 将所有声道混合为单声道（取平均值） |
| Stereo | 如果源是单声道，复制为双声道；如果源已是立体声，保持不变 |

> 💡 **提示**：对于语音内容，选择 Mono 可以显著减小文件大小（约 50%），且不影响可懂度。

## 6. 音频处理参数详解

转换器在处理音频时按照固定的 **7 步顺序**执行：

```
Step 1: 声道转换 (Channels)
    ↓
Step 2: 音量调节 (Volume)
    ↓
Step 3: 重采样 + 速度/音高 (Sample Rate / Speed / Pitch)
    ↓
Step 4: 淡入淡出 (Fade In / Fade Out)
    ↓
Step 5: 降噪 (Noise Reduction)
    ↓
Step 6: 峰值归一化 (Normalize Peak)
    ↓
Step 7: 编码输出 (Output Format)
```

> ⚠️ **重要**：处理顺序是固定的，不可调整。例如，音量调节在重采样之前执行，这意味着如果你同时调整了音量和采样率，音量调整会先作用于原始采样率的音频，然后再进行重采样。

### 6.1 Volume（音量）

- **范围**：0% – 200%
- **默认值**：100%（无变化）
- **说明**：在导出前对音频幅度进行缩放。0% = 完全静音，200% = 双倍振幅。
- **削波保护**：如果音量超过 100%，且源音频已经接近 0dBFS（最大电平），可能会产生数字削波失真。建议配合 **Normalize Peak** 使用，或在应用高音量前先降低原始电平。

### 6.2 Speed（速度）

- **范围**：25% – 400%
- **默认值**：100%（无变化）
- **说明**：调整音频的播放速度。使用 Web Audio API 的 `playbackRate` 参数配合 `OfflineAudioContext` 实现时间伸缩。
- **音高独立性**：默认情况下，Speed 参数会同时改变速度和音高（与播放器的变速播放相同）。如果你希望只改变速度而保持音高，需要将 **Pitch** 参数设置为相反方向的补偿值。例如，Speed = 200%（升高一个八度），则 Pitch = -1200 cents（降低一个八度）。

### 6.3 Pitch（音高）

- **范围**：-1200 cents 至 +1200 cents
- **默认值**：0
- **说明**：以 cents（音分）为单位调整音高。100 cents = 1 个半音，1200 cents = 1 个八度。
- **使用场景**：
  - 微调跑调的录音：±50 cents
  - 改变歌曲调性：±200 ~ ±700 cents
  - 创造特殊效果：±1200 cents（一个八度上下）
- **与 Speed 的关系**：Pitch 使用 Web Audio API 的 `detune` 参数，它相对于当前 playbackRate 调整音高。如果 Speed = 100% 且 Pitch = 0，音高不变。如果 Speed = 200% 且 Pitch = 0，音高会自动升高一个八度。要抵消 Speed 的音高变化，需要设置反向的 Pitch 值。

### 6.4 Fade In（淡入）

- **范围**：0 – 10 秒
- **默认值**：0
- **说明**：在音频开头应用线性音量渐变，从 0% 逐渐上升到 100%。
- **推荐值**：
  - 语音：0.1 – 0.5 秒（避免喷麦声）
  - 音乐：0.5 – 2 秒（自然引入）
  - 音效：0.05 – 0.2 秒

### 6.5 Fade Out（淡出）

- **范围**：0 – 10 秒
- **默认值**：0
- **说明**：在音频结尾应用线性音量渐变，从 100% 逐渐下降到 0%。
- **推荐值**：
  - 语音：0.2 – 1 秒
  - 音乐：1 – 5 秒（根据混响尾音长度调整）
  - 循环音频：0 秒（避免循环点处音量突变）

> 💡 **淡入淡出重叠**：如果音频时长很短，导致淡入和淡出区域重叠，转换器会使用**乘法叠加**方式计算增益。例如，在重叠区域的某点，fade-in 增益为 0.5，fade-out 增益为 0.8，则最终增益 = 0.5 × 0.8 = 0.4。

### 6.6 Noise Reduction（降噪）

- **范围**：0% – 100%
- **默认值**：0（关闭）
- **说明**：使用简单的门限降噪算法。降噪强度决定了噪声门限的高度：
  - 门限 = (intensity / 100) × 0.02
  - 低于门限的样本会被衰减（衰减比例与该样本的振幅成正比）
  - 高于门限的样本保持不变
- **推荐值**：
  - 轻度降噪：10 – 20%（去除轻微底噪）
  - 中度降噪：30 – 50%（去除明显的环境噪声）
  - 重度降噪：60 – 100%（去除强噪声，但可能损失音频细节）

> ⚠️ **警告**：过高的降噪强度（>70%）会导致音频失真、出现"水声"伪影，并可能消除安静的语音细节。建议从低强度开始逐步增加。

### 6.7 Normalize Peak（峰值归一化）

- **类型**：开关（Toggle Chip）
- **默认值**：关闭
- **说明**：当启用时，转换器会在所有其他处理完成后，分析音频的最大绝对值（峰值），并将整个音频缩放至峰值达到 99.9% 的满刻度（0dBFS）。
- **与 Volume 的区别**：
  - Volume 是按你指定的固定比例缩放
  - Normalize Peak 是自动计算最优缩放比例，使音频尽可能响亮但不削波
- **适用场景**：
  - 批量统一多个音频的响度
  - 修复录制过轻的音频
  - 准备用于广播/发布的音频

> 💡 **Normalize Peak 的执行顺序**：在所有其他处理（音量、重采样、淡入淡出、降噪）**之后**执行。这意味着如果你同时设置了 Volume = 200% 和 Normalize Peak = 开启，最终峰值仍然会被限制在 99.9%，因为 Normalize 是最后一步。

## 7. 执行转换

### 7.1 开始转换

设置好所有参数后，点击 **「Convert」** 按钮。按钮在以下情况下会被禁用：

- 未导入音频（`!sourceBuffer`）
- 正在转换中（`isConverting`）

### 7.2 转换进度

转换过程中，按钮文字会变为 `Converting XX%…`，同时下方显示进度条：

```
████████████████████████████░░░░ 75%
```

进度条对应 7 步处理链的进度：

| 进度 | 处理步骤 | 说明 |
|------|----------|------|
| 5% | 初始化 | 验证参数、准备缓冲区 |
| 12% | 声道转换 | Mono/Stereo 转换完成 |
| 20% | 音量调节 | Volume 应用完成 |
| 40% | 重采样 | Speed/Pitch/Rate 处理完成（最耗时的一步） |
| 55% | 淡入淡出 | Fade 应用完成 |
| 65% | 降噪 | Noise Reduction 应用完成 |
| 75% | 峰值归一化 | Normalize 完成 |
| 100% | 编码输出 | 文件生成完成 |

### 7.3 转换时间估算

转换时间主要取决于：

1. **音频时长**：越长越慢
2. **重采样**：改变采样率或速度/音高时最耗时
3. **输出格式**：WAV 格式编码很快；MediaRecorder 格式（WebM/OGG/MP3/MP4）需要实时播放再录制，速度约等于音频时长

| 场景 | 大致耗时 |
|------|----------|
| 3 分钟音频 → WAV，仅改音量 | < 1 秒 |
| 3 分钟音频 → WAV，重采样 + 速度 | 2 – 5 秒 |
| 3 分钟音频 → MP3/OGG（MediaRecorder）| 3 – 4 分钟（实时录制） |

> ⏳ **MediaRecorder 格式的等待**：选择 WebM/Opus、OGG/Opus、MP3 或 MP4/AAC 时，转换器需要通过浏览器的 MediaRecorder API 实时"播放"音频并录制输出。这意味着转换 3 分钟的音频大约需要等待 3 分钟。进度条会在 Step 7（编码输出）阶段缓慢增长。

### 7.4 转换失败

如果转换过程中发生错误：

1. 进度条停止增长
2. 按钮恢复为「Convert」
3. 错误信息会显示在日志面板中
4. 如果设置中启用了错误面板（默认开启），会弹出 DraggableErrorPanel，显示错误代码、消息、原因和解决方案
5. 点击错误面板中的 **「Retry」** 可以重试转换

## 11. 故障排除指南

### 问题 1：导入时提示 "Failed to decode audio file"

**可能原因**：
- 文件不是有效的音频文件
- 文件使用了浏览器不支持的编码（如某些特殊的 AAC Profile）
- 文件已损坏

**解决方案**：
1. 在系统播放器中确认文件可以正常播放
2. 尝试将文件转换为 WAV 格式后再导入
3. 检查文件扩展名是否与实际编码匹配

### 问题 2：选择 MP3/OGG 输出格式后转换失败

**可能原因**：当前浏览器不支持该格式的 MediaRecorder。

**解决方案**：
1. 切换到 **WAV 16-bit PCM**，这是所有浏览器都支持的格式
2. 尝试在其他浏览器中操作（Chrome 通常支持最全面的 MediaRecorder 格式）
3. 检查浏览器版本是否过旧

### 问题 3：转换后的音频音质很差

**可能原因**：
- 从无损格式转换到了有损格式（如 WAV → MP3）
- 选择了过低的位深（如 8-bit）
- 降噪强度过高导致细节损失

**解决方案**：
1. 如果需要保持音质，选择 WAV 16-bit 或 24-bit
2. 降噪强度控制在 30% 以下
3. 避免多次有损转换（如 MP3 → OGG → MP3）

### 问题 4：转换后的文件大小比原文件大很多

**可能原因**：
- 从压缩格式（MP3/OGG）转为了无损格式（WAV）
- 升采样到更高的采样率（如 44100Hz → 192000Hz）
- 从单声道转为了立体声

**解决方案**：
1. 如果需要小文件，选择压缩输出格式（WebM/Opus 压缩率最高）
2. 保持原始采样率
3. 对于语音内容，选择 Mono 声道

### 问题 5：进度条卡在 75% 不动

**可能原因**：MediaRecorder 格式编码时，进度条在最后阶段更新较慢。

**解决方案**：
1. 耐心等待，MediaRecorder 需要实时"播放"完整段音频
2. 如果超过音频时长的 2 倍时间仍无响应，可能是浏览器卡死，请刷新页面

## 12. 常见问题 FAQ

**Q1：音频格式转换器支持批量转换多个文件吗？**

A：当前版本（v1.7.1）每次只能处理一个文件。你可以依次导入不同的文件进行转换。批量转换功能计划在后续版本中加入。

**Q2：转换后的 FLAC 文件为什么实际上是 WAV？**

A：浏览器没有内置的 FLAC 编码器。当你选择 FLAC 输出时，转换器会自动回退到 WAV 16-bit PCM 格式，并在日志中记录回退信息。

**Q3：我可以在转换过程中关闭页面吗？**

A：不可以。所有处理都在浏览器内存中进行，关闭页面会丢失所有未下载的结果。

**Q4：转换后的 MP3 音质可以调整吗？**

A：当前版本通过 MediaRecorder API 生成 MP3，比特率由浏览器控制，无法手动设置。如果需要精确的比特率控制，建议先用转换器输出 WAV，再使用专业的音频编码软件进行压缩。

**Q5：速度和音高可以独立调整吗？**

A：可以，但需要手动补偿。Speed 参数会同时改变速度和音高。如果你只想改变速度而保持音高，需要同时设置反向的 Pitch 值。例如：Speed = 50%（慢速，音高低八度），Pitch = +1200 cents（升高八度），最终效果是速度减半，音高不变。

**Q6：降噪功能能完全消除背景噪声吗？**

A：不能完全消除。转换器使用的是简单的门限降噪算法，适合去除持续的低电平噪声（如风扇声、电流声）。对于复杂的噪声（如多人说话、音乐背景），效果有限。建议配合音频编辑器使用更高级的降噪效果。

**Q7：转换后的音频在手机上无法播放？**

A：某些手机播放器可能不支持 WAV 24-bit 或 32-bit Float。如果需要在手机上播放，建议使用 WAV 16-bit PCM 或 MP3 格式。

**Q8：Normalize Peak 和 Volume 会冲突吗？**

A：不会冲突，但有先后顺序。Volume 先执行，Normalize Peak 最后执行。如果你同时设置了高音量和开启 Normalize，最终峰值仍然会被限制在 99.9%。

## 13. 技术规格

### 13.1 音频引擎

| 项目 | 规格 |
|------|------|
| 解码 API | Web Audio API `AudioContext.decodeAudioData()` |
| 处理 API | Web Audio API `OfflineAudioContext` |
| 压缩编码 | MediaRecorder API（浏览器原生） |
| WAV 编码 | 自定义 JavaScript 编码器（RIFF/WAVE） |
| 采样精度 | 32-bit Float（内部处理） |

### 13.2 WAV 导出格式详情

| 位深 | 编码 | 声道 | 头部 | 数据格式 |
|------|------|------|------|----------|
| 8-bit | 无符号整数 | 支持 1-2ch | RIFF/WAVE | PCM (wFormatTag = 1) |
| 16-bit | 有符号整数 | 支持 1-2ch | RIFF/WAVE | PCM (wFormatTag = 1) |
| 24-bit | 有符号整数 | 支持 1-2ch | RIFF/WAVE | PCM (wFormatTag = 1) |
| 32-bit | IEEE 浮点 | 支持 1-2ch | RIFF/WAVE | FLOAT (wFormatTag = 3) |

### 13.3 降噪算法

```
门限 = (intensity / 100) × 0.02
if |sample| < 门限:
    sample = sample × (|sample| / 门限)
else:
    sample = sample（不变）
```

### 13.4 淡入淡出算法

```
fadeInGain(i) = i / (fadeInSeconds × sampleRate)
fadeOutGain(i) = (totalSamples - i) / (fadeOutSeconds × sampleRate)
totalGain(i) = fadeInGain(i) × fadeOutGain(i)  // 乘法叠加
```

### 13.5 峰值归一化算法

```
peak = max(|sample|) for all channels and samples
if peak > 0 and peak < 0.999:
    gain = 0.999 / peak
    all samples *= gain
```

### 13.6 浏览器兼容性

| 浏览器 | WAV | WebM/Opus | OGG/Opus | MP3 | MP4/AAC |
|--------|-----|-----------|----------|-----|---------|
| Chrome | ✅ | ✅ | ✅ | ✅ | ✅ |
| Firefox | ✅ | ✅ | ✅ | ❌ | ❌ |
| Safari | ✅ | ❌ | ❌ | ✅ | ✅ |
| Edge | ✅ | ✅ | ✅ | ✅ | ✅ |

## 14. 高级技巧

### 技巧 1：快速统一多个音频的响度

如果你有一系列录制音量不一致的音频文件，可以：

1. 导入第一个文件
2. 开启 **Normalize Peak**
3. 保持其他参数为默认值
4. 转换并下载
5. 点击 **Replace** 导入下一个文件
6. 重复步骤 3-5

所有输出的音频都会具有相同的峰值电平（99.9%）。

### 技巧 2：创建慢速不降低音高的效果

1. 设置 **Speed** = 50%
2. 设置 **Pitch** = +1200 cents
3. 转换

效果：音频时长加倍，但音高保持不变。适合制作慢动作视频的背景音乐。

### 技巧 3：批量去除音频底噪

如果你有一批录音都有类似的底噪（如电流声）：

1. 导入一个有代表性的文件
2. 逐步增加 **Noise Reduction** 强度（10% → 20% → 30%）
3. 每次转换后预览，找到最佳平衡点
4. 确定最佳强度后，对所有文件使用相同的参数

### 技巧 4：从立体声提取单声道

1. 导入立体声音频
2. 设置 **Channels** = Mono
3. 转换

文件大小会减半，适合只需要单声道内容的场景（如播客、语音备忘录）。

### 技巧 5：为视频制作音频轨道

视频编辑软件通常要求 48000Hz 采样率：

1. 导入音频
2. 设置 **Sample Rate** = 48000 Hz
3. 选择适合的输出格式（WAV 16-bit 最通用）
4. 转换并导入视频编辑软件

### 技巧 6：创建 8-bit 复古音效

1. 导入现代音频
2. 设置 **Output Format** = WAV 8-bit PCM
3. 可选：降低 **Sample Rate** 到 22050 Hz
4. 转换

效果：明显的量化噪声和低保真质感，适合复古游戏风格。

## 15. 已知限制与注意事项

### 15.1 FLAC 编码限制

浏览器没有原生的 FLAC 编码器。选择 FLAC 输出时，实际得到的是 WAV 16-bit PCM 文件。如果你确实需要 FLAC 格式，请先用转换器输出 WAV，再使用外部工具（如 Audacity、ffmpeg）转换为 FLAC。

### 15.2 MediaRecorder 格式限制

WebM/Opus、OGG/Opus、MP3、MP4/AAC 的输出质量由浏览器的 MediaRecorder 实现决定，**无法手动设置比特率**。不同浏览器的输出质量可能有显著差异：

- Chrome 的 WebM/Opus 通常质量较好
- Safari 的 MP4/AAC 质量较好
- Firefox 的 OGG/Opus 质量较好

### 15.3 内存限制

所有音频数据都存储在浏览器内存中。处理超长音频（>1 小时）或高采样率（192000Hz）的多声道音频时，可能会超出浏览器的内存限制，导致页面崩溃。建议分段处理长音频。

### 15.4 单声道 → 立体声

将单声道转换为立体声时，转换器只是将相同的信号复制到左右两个声道，**不会创造真正的立体声场**。如果需要真正的立体声效果（如声像、混响），请使用音频编辑器。

### 15.5 多次有损转换

避免多次在不同有损格式之间转换（如 MP3 → OGG → MP3）。每次有损转换都会进一步降低音质。如果需要中间处理，始终使用无损格式（WAV）作为中转。

### 15.6 重采样质量

`OfflineAudioContext` 的重采样算法由浏览器实现，质量通常足够好，但不是最高质量的（如 SoX 的 VHQ 模式）。对于专业母带处理中的重采样，建议使用专门的音频软件。

## 16. 与音频编辑器的协作

音频格式转换器和音频编辑器可以无缝协作：

### 从转换器跳转到编辑器

在转换器页面标题栏点击 **「🎵 Audio Editor」** 按钮，即可直接跳转到音频编辑器页面。跳转后：

- 当前转换器的设置和结果不会保留（两个页面是独立的）
- 你可以在编辑器中进行更高级的波形编辑

### 建议的工作流

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| 只需要转换格式/采样率 | 音频格式转换器 | 更快捷，界面简洁 |
| 需要剪辑音频 | 音频编辑器 | 支持波形级精确剪辑 |
| 需要应用多种效果 | 音频编辑器 | 支持 EQ、压缩器、混响等实时效果 |
| 需要批量标准化响度 | 音频格式转换器 | Normalize Peak 一键完成 |
| 需要降噪 + 剪辑 | 两个工具配合使用 | 先用编辑器剪辑，再用转换器降噪导出 |

### 数据互通

当前版本（v1.7.1）两个工具之间**不共享音频数据**。如果你希望在编辑器中继续处理转换器中的音频，需要：

1. 在转换器中完成转换
2. 点击 Download 保存到本地
3. 跳转到音频编辑器
4. 在编辑器中重新导入刚才下载的文件
